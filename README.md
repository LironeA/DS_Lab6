# Лабораторна робота №6. Atomic Swap

Консольний застосунок на C# (.NET), який демонструє навчальну симуляцію тристороннього `Atomic Swap` між `Party A`, `Party B` і `Party C`.

## Що таке Atomic Swap

`Atomic Swap` це обмін активами між сторонами за правилом "або виконуються всі взаємопов'язані перекази, або ніхто не втрачає кошти". У лабораторній це моделюється через умовні контракти і таймлоки.

## Що таке HTLC

`HTLC (Hashed Timelock Contract)` це контракт, який має дві ключові умови:

- `Hash Lock`: актив можна отримати лише після надання правильного секрету `S`, який відповідає `hash(S)`.
- `Time Lock`: якщо секрет не розкрито до завершення часу дії контракту, відправник може виконати `refund`.

## Модель з 3 процесами

У проєкті використовуються окремі процеси:

- `Coordinator` готує сценарій, створює спільну теку `runs/...`, запускає процеси `PartyA`, `PartyB`, `PartyC`, чекає завершення і формує summary.
- `PartyA` генерує секрет, обчислює `SHA256(secret)`, створює та фінансує контракт `A -> B`.
- `PartyB` чекає контракт `A -> B`, створює та фінансує контракт `B -> C`.
- `PartyC` чекає контракт `B -> C`, створює та фінансує контракт `C -> A`.

Обмін між процесами зроблено максимально просто через JSON-файли:

- `runs/<scenario>/state/scenario.json`
- `runs/<scenario>/state/balances.json`
- `runs/<scenario>/state/contract_ab.json`
- `runs/<scenario>/state/contract_bc.json`
- `runs/<scenario>/state/contract_ca.json`
- `runs/<scenario>/logs/*.log`

## Реалізовані сценарії

### 1. Successful three-party atomic swap demo

- `PartyA` генерує секрет і хеш.
- Всі три контракти використовують один і той самий `hash lock`.
- `PartyA` першим робить `redeem` у контракті `C -> A` і тим самим розкриває секрет.
- `PartyC` бачить секрет і робить `redeem` у контракті `B -> C`.
- `PartyB` бачить той самий секрет і робить `redeem` у контракті `A -> B`.
- Усі сторони отримують очікувані активи.

### 2. Timeout / refund demo

- Контракти створюються і фінансуються так само.
- `PartyA` навмисно не виконує `redeem`.
- Секрет не розкривається.
- Після завершення таймлоків кожна сторона виконує `refund` для свого вихідного контракту.
- Баланси повертаються у початковий стан.

## Меню застосунку

Після запуску без аргументів з'являється меню:

1. `Run successful three-party atomic swap demo`
2. `Run timeout/refund demo`
3. `Show final logs`
4. `Exit`

## Запуск

Потрібно мати встановлений `.NET 8 SDK`.

```bash
dotnet run
```

Або окремо:

```bash
dotnet build
dotnet run -- coordinator Success
dotnet run -- coordinator Timeout
dotnet run -- showlogs
```

## Логування

Кожний процес веде власний лог-файл у форматі:

```text
[2026-03-29 21:10:01.120] [PartyA] [INFO] Generated secret and hash lock
```

Логуються такі події:

- запуск процесу;
- створення контракту;
- funding;
- очікування інших подій;
- перевірка `hash lock`;
- `redeem`;
- розкриття секрету;
- `refund`;
- завершення процесу;
- фінальні баланси і стани контрактів.

## Навчальний характер моделі

Це не реальний блокчейн і не робота з криптовалютою. У проєкті використано:

- умовні баланси;
- умовні HTLC-контракти;
- `SHA256` для хешування;
- `DateTime.UtcNow` для таймлоків;
- файловий обмін станом через JSON.

Головна мета проєкту це наочно показати принцип роботи `Atomic Swap` і `HTLC` у розподіленій системі.
